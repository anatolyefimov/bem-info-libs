﻿Блок ##page## создаёт теги верхнего уровня страницы:

Именно он отвечает за то, какие подключить ##CSS## и ##JS## файлы к странице, выставление ##meta## тегов, заголовка и так далее.

Декларация блока в ##BEMJSON## начинается объявлением блока и указанием свойства ##title##, которое превращается в тег #### в ##HTML##.</p> <p>%%hl js ({ block: 'page', title: 'Page with link', ... }) %%</p> <p>Указание свойства ##head## дополняет элемент ##head##, соответствующий ##HTML## тегу ##<head>##, элементами для подключения ##CSS## и ##JS## файлов, а также указания ##meta##:</p> <p>%%hl js ({ ... head: [ { elem: 'css', url: 'example.css', ie: false }, { elem: 'css', url: 'example.ie.css', ie: 'lt IE 8' }, { elem: 'js', url: 'example.js' }, { elem: 'meta', attrs: { name: 'keywords', content: 'js, css, html' } }, { elem: 'meta', attrs: { name: 'description', content: 'Yet another webdev blog' } } ], ... }) %%</p> <p>Элемент ##css## превращается в ##HTML## в тег ##<link>##, подключающий как ##CSS## стиль тот файл, что указан в свойстве ##url## этого элемента. Также у такого элемента может быть свойство ##ie##. Если это свойство ##false##, то будут использоваться такие ##condittional comments##, которые предотвратят использование этих стилей в ##IE##. При строчном значении этого свойства тег ##<link>## , будет обёрнут в соответствующий ##conditional comment##, и этот стиль будет грузиться и использоваться указанных браузерах.\ Также есть возможность указывать свойство ##content## для содержания тега ##<style>##:</p> <p>%%hl js ({ ... head: [ { elem: 'css', content: '.b-blah { color: #f00' } }, ... ], ... }) %%</p> <p>Элемент ##js## действует аналогично, подключая к странице ##JS## файлы при помощи тега</p> <h2 id="-script-"><script>##.</h2> <p>Свойство ##head## не описывает содержание ##HTML##-тега ##<head>## полностью, а лишь дополняет дефолтное, которое блок сам создаёт в своём ##BEMHTML## шаблоне.</p> <p>==== Тег <meta> с указанием кодировки</p> <h2 id="bemhtml-">BEMHTML##:</h2> <p>%%hl js content: [ { tag: 'meta', attrs: { 'http-equiv': 'content-type', content: 'text/html; charset=utf-8' } }, ... %%</p> <p>==== Тег <meta> для использования ##IE9## (и выше) в максимальном ##compatibility## режиме</p> <h2 id="bemhtml-">BEMHTML##:</h2> <p>%%hl js content: [ ... { tag: 'meta', attrs: { 'http-equiv': 'X-UA-Compatible', content: 'IE=EmulateIE7, IE=edge' } }, ... %%</p> <p>==== Выставление значения тега <title> страницы из свойства</p> <p>%%hl js content: [ ... { tag: 'title', content: this.ctx.title }, ... %%</p> <p>==== Выставление фавиконки</p> <p>%%hl js content: [ ... this.ctx.favicon ? { elem: 'favicon', url: this.ctx.favicon } : '', ... %%</p> <p>==== Декларация блока ua</p> <p>%%hl js content: [ ... { block: 'ua' }, ... %%</p> <p>Значением свойства ##content## блока ##page## может быть хеш-описание содержимого (если речь идёт лишь об одном блоке) или массив блоков, описанных хешами:</p> <p>%%hl js ({ ... content: { block: 'b-link', mods: { pseudo: 'yes', togcolor: 'yes', color: 'green' }, url: '#', target: '_blank', title: 'Кликни меня', content: 'Псевдоссылка, меняющая цвет по клику' } }) %%</p> <p>На блоки, содержащиеся в ##content##, действуют их ##BEMHTML## шаблоны.</p> <p>==== Отмена автоматической инициализации блоков</p> <p>%%hl js noDeps: [ { block: 'i-bem', elem: 'dom', mods: { init: 'auto' } } ] %%</p> </div></div></div></body></html>
